# Contributing to Tigray Knowledge Base

Thank you for your interest in contributing! We welcome contributions to content, code, data, and design.

## How to Contribute

1. Fork the repository
2. Create a feature branch: `git checkout -b feature/brief-description`
3. Make changes and test locally (`mkdocs serve`)
4. Commit with descriptive messages and sign off if appropriate
5. Push to your fork and open a Pull Request against `main`

## Code Standards
- Follow Python and JavaScript best practices
- Keep changes small and focused
- Add or update tests where applicable

## Content Guidelines
- Cite reliable sources
- Keep an objective, neutral tone
- Respect privacy and legal restrictions

## Issues & Support
- Report bugs or request features via GitHub Issues
- For discussion, use GitHub Discussions

## Contact
If you need help, open an issue or reach out via the project contact links in the README.
